PrivyPrompt Privacy Policy

Last updated: July 3, 2026

PrivyPrompt detects and protects sensitive information before users submit prompts or supported files to generative AI services. It also checks locally for AI tool security risks such as prompt-injection, jailbreak, system-prompt leakage, local-secret access, and secret-exfiltration patterns.

Local processing

  • Prompt text and supported files are processed locally inside the browser.
  • AI tool security checks are local pattern checks and do not send prompts to an external AI review service.
  • PrivyPrompt does not transmit prompts, files, detected values, custom terms, private aliases, or AI responses to the developer or advertising services.
  • PDF.js, Mammoth, and Tesseract.js assets are bundled with the extension and do not load remote code at runtime.

Local storage

PrivyPrompt stores protection preferences, selected country packs, user-created protected terms, and aggregate scan counters in local extension storage. It does not intentionally save complete prompts or detected sensitive values.

Website access

Website access is limited to supported AI services so PrivyPrompt can inspect user-initiated prompts and file selections immediately before submission and display local protection controls.

Private aliases and response restoration

Placeholder-to-original mappings exist only in the current tab's extension memory and disappear when the page closes or reloads. Restored originals are displayed in a closed extension-created view and are not inserted into the AI site's normal page DOM.

Network activity

PrivyPrompt does not make external network requests for analytics, advertising, prompt processing, document processing, or OCR.

Data selling and advertising

PrivyPrompt does not sell user data and does not use user data for advertising.

Changes

This policy will be updated if PrivyPrompt's data handling changes. Any future remote-processing capability will require clear disclosure and user consent before activation.
